#보이스피싱 및 스피어피싱 탐지 시스템 연구

## 1. 서론 (Introduction)
보이스피싱 범죄가 급증하면서 이를 대응하기 위한 기술 연구가 활발히 진행되고 있습니다. 그러나 기존 공개 보이스피싱 데이터와 일반 데이터는 개인정보 처리자 도메인에 대한 탐지 성능이 낮아, 대규모로 유출되는 스피어피싱 공격에 적용 가능한 기술은 부족합니다.
본 연구에서는 이러한 문제를 해결하기 위해, LM 기반 데이터 확장과 맥락 기반 탐지 모델을 결합한 시스템을 설계하고 구현했습니다. 또한, 실시간 통화 환경에서의 발화 차단이 불가능한 현실적 제약을 고려하여, 온디바이스 환경에서 경량화 모델로 구현 가능한 구조를 제안합니다.

## 2. 관련 연구 (Related Work)
보이스피싱 탐지: 기존 연구는 주로 음성 합성 탐지 또는 STT 기반 텍스트 분석에 집중
데이터 한계 문제: 공개 데이터는 개인정보 도메인을 충분히 반영하지 못함
LLM 활용 연구: 최근 언어 모델을 통한 데이터 생성 및 도메인 확장 연구가 증가
하지만 기존 연구는 실제 통화 환경 재현과 온디바이스 구현 측면에서 한계가 있으며, 스피어피싱 도메인에 대한 탐지 성능 검증도 부족합니다.

## 3. 연구 문제 정의 (Problem Definition)
본 연구에서 해결하고자 하는 문제는 다음과 같습니다:
기존 데이터셋으로는 개인정보 처리자 도메인 탐지 성능 저하
실시간 전화 환경에서 발화 차단 불가능
서버 기반 LLM 사용 시 발생하는 지연(latency), 데이터 보안, 법적/심리적 부담
이에 따라 연구 목표는 다음과 같이 설정했습니다:
스피어피싱 대응 가능 보이스피싱 탐지 모델 개발
전화 음성 기반 화자 검증 및 맥락 기반 탐지
온디바이스 환경에서의 모델 경량화 및 최적화
현실적 전화 환경 강건성 검증

## 4. 연구 방법 (Methodology)

### 4.1 데이터 확장 및 생성
LM 기반 텍스트 데이터 생성
시나리오 및 프롬프트 엔지니어링 적용
생성 데이터는 실제 보이스피싱 환경과 유사하게 설계

### 4.2 시스템 구조
전화 음성 → 딥보이스 모델 & Google STT
   ├─ 딥보이스: 음성 합성/위조 여부 판단
   └─ STT: 음성 → 텍스트 변환 → 맥락 기반 탐지 모델
화자 검증: 딥보이스 모델을 통한 음성 합성/위조 여부 판단
맥락 검증: STT 텍스트를 기반으로 LSTM 모델을 통한 보이스피싱 탐지
온디바이스 실행: 서버 통신 없이 경량화 모델로 실행 가능

### 4.3 모델 설계 및 최적화
맥락 기반 판단 모델: 경량화 LSTM 사용
모델 용량, 파라미터 수, 연산 리소스 최적화
최적 파라미터 도출 및 성능 확보

### 4.4 서비스 환경 구성
VOIP 테스트 환경: AWS EC2 + Asterisk
실제 전화 환경을 반영하여 STT 손실률, 잡음 등 재현
모델 학습, 결과, 데이터 생성 및 파이프라인 설계 포함
실제 서비스 환경과 유사한 시연 영상 제공

## 5. 실험 (Experiments)
LM 기반 데이터 학습 모델의 실제 데이터 적응력 검증
STT 손실률, 전화 환경 제한을 반영한 모델 강건성 실험
다양한 LSTM 하이퍼파라미터 실험을 통한 최적 모델 확보

## 6. 결과 (Results)
생성 데이터 기반 학습 모델이 실제 데이터에서도 높은 적응력과 커버리지 확보
STT 손실률과 전화 환경 제약에도 강건성 검증 성공
기존 공개 서비스 대비 재현 가능성 및 연구적 기여 확보

## 7. 토의 (Discussion)
LM을 통한 데이터 생성은 도메인 확장에 효과적
화자 검증 + 맥락 기반 탐지 구조는 실제 서비스 적용 가능성 높음
온디바이스 경량화 모델은 사용자 프라이버시와 법적 리스크 최소화
현재 통신사 환경에서는 연구 재현이 어려움 → 본 연구의 실용적 가치 강조

## 8. 결론 (Conclusion)
본 연구는 LM 기반 데이터 확장과 경량화 LSTM 모델을 결합하여, 스피어피싱 대응 가능 보이스피싱 탐지 시스템을 제안했습니다.
실험 결과, 실제 통화 환경에서도 높은 성능과 강건성을 확인했으며, 온디바이스 환경 구현과 재현 가능한 연구 구조를 통해 향후 연구 확장 가능성을 확보했습니다.
